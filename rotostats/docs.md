## User-Visible API Changes

### New exported function: `inverse_categories()`

```r
# R/inverse-categories.R
inverse_categories()
#> [1] "ERA"  "WHIP" "FIP"  "xFIP" "SIERA" "xERA" "BB/9" "HR/9"
```

Returns the package-level list of lower-is-better scoring categories (pitcher
ratio stats where lower totals are better). This is the Layer-3 fallback used
by `sgp_denominators()`.

### New parameter: `league_config(inverse_categories = NULL)`

```r
# Declare directionality once at the league level
cfg <- league_config(
  n_teams            = 12L,
  roster_slots       = c(C = 1L, "1B" = 1L, OF = 3L),
  categories         = c("HR", "R", "RBI", "ERA", "WHIP", "FIP"),
  inverse_categories = c("ERA", "WHIP", "FIP")   # NEW
)
cfg$inverse_categories
#> [1] "ERA"  "WHIP" "FIP"

print(cfg)
# ... (existing lines) ...
#   Inverse:    ERA WHIP FIP
```

- `NULL` (default): no league-level override; downstream consumers fall through
  to the package lookup.
- Valid character vector: normalized to uppercase, validated against
  `categories`, stored in the S3 object.
- Invalid inputs abort with `rotostats_error_invalid_inverse_categories`.

### Changed default and new parameter: `sgp_denominators()`

**Default change:** `inverse_categories` default changed from `c("ERA","WHIP")` to `NULL`.

**New parameter:** `config = NULL` — accepts a `league_config` object for Layer-2 resolution.

```r
# Layer-3 (package default) — no change in behavior for ERA/WHIP leagues:
d <- sgp_denominators(lh, scoring_categories = c("ERA", "WHIP"))
# Effective inverse categories: 'ERA', 'WHIP'. (package default)

# Layer-3 with FIP scored — FIP automatically direction-flipped:
d <- sgp_denominators(lh, scoring_categories = c("ERA", "WHIP", "FIP"))
# Effective inverse categories: 'ERA', 'FIP', 'WHIP'. (package default)

# Layer-2 — inherit from config:
cfg <- league_config(
  n_teams            = 12L,
  roster_slots       = c(C = 1L),
  categories         = c("ERA", "WHIP"),
  inverse_categories = c("ERA")
)
d <- sgp_denominators(lh, scoring_categories = c("ERA", "WHIP"), config = cfg)
# Effective inverse categories: 'ERA'. (from config)

# Layer-1 — explicit user override (full replacement; config ignored):
d <- sgp_denominators(
  lh,
  scoring_categories = c("ERA", "WHIP", "FIP"),
  inverse_categories = "FIP",    # only FIP; ERA and WHIP NOT flipped
  config             = cfg
)
# Effective inverse categories: 'FIP'. (user override)
```

**Three-layer resolution order:**

1. `inverse_categories` non-NULL (explicit arg) → use it (full replacement,
   validate against `scoring_categories`)
2. `config` non-NULL AND `config$inverse_categories` non-NULL → inherit from
   config verbatim (already validated at construction)
3. All else → `intersect(scoring_categories, inverse_categories())`
   (package default, preserves ERA/WHIP legacy behavior)
