# 2026-04-18 — sgp_denominators() inverse_categories Parameter

> Run: `sgp-denom-inverse-categories-param-2026-04-17` | Profile: r-package | Verdict: PASS

## What Changed

Replaced the hard-coded package-level constant `INVERSE_CATEGORIES <- c("ERA", "WHIP")` in
`R/sgp-denominators-helpers.R` with a new user-facing argument `inverse_categories` on
`sgp_denominators()`. The default value is unchanged (`c("ERA", "WHIP")`), so all existing callers
are unaffected. Users can now declare additional lower-is-better scoring categories (OAVG, BB9, etc.)
without modifying package source. A one-shot `cli_inform()` per call reports the effective
direction-flip set. A BLOCK in tester round 1 revealed that the original strict membership validation
broke batting-only leagues; the fix introduces a `missing()`-based default-vs-explicit distinction.

## Design Decisions

1. **`missing()` over `NULL` sentinel for default detection**: The user's hard constraint requires the default literal in the function signature to be `c("ERA", "WHIP")` so that (a) `man/sgp_denominators.Rd` shows the real default, not `NULL`, and (b) callers who write `inverse_categories = c("ERA", "WHIP")` explicitly continue to work. Using `NULL` as a sentinel would have required changing the published interface. `missing()` achieves default detection without any signature change. It is a standard R idiom with no `R CMD check` implications, provided the flag is captured as the very first statement of the function body before any other code references the argument.

2. **Content-addressed `.frequency_id` without `digest` dependency**: The rlang `.frequency = "once"` mechanism requires `.frequency_id` (rlang >= 1.1.0). The chosen id is `paste0("rotostats_sgp_denom_inverse_", paste(sort(inverse_categories), collapse = ","))`. This is content-addressed: identical configurations share a key (suppressed after first call in a session); distinct configurations get different keys (each announces itself once). Sorting the categories before pasting ensures `c("ERA", "WHIP")` and `c("WHIP", "ERA")` produce the same id. No external `digest` dependency is needed; pure base R string operations are sufficient given that category names are short ASCII strings.

3. **Default-path silent intersection as the correct backward-compatible semantics**: The original `INVERSE_CATEGORIES` constant operated via `%in%` at runtime, which is equivalent to a set intersection: only categories that are both in the constant AND in the per-loop `cat` values have any effect. A batting-only loop never encountered a category in `c("ERA", "WHIP")`, so the constant had no effect. Replacing the constant with a validated argument that aborts on unrecognized elements is correct for user-supplied values (where the abort surfaces a likely mistake) but incorrect for the default (where the abort would block batting-only leagues that were working fine before). The silent intersection on the default path faithfully reproduces the original behavior.

4. **Four threading sites (not three)**: The spec summary listed three `INVERSE_CATEGORIES` usage sites in `R/sgp-denominators.R` (lines ~623, ~730, ~739, ~886). The implementation replaced all four occurrences. The slope-sign check at lines ~730 and ~739 is a two-branch structure (if/else if), so each branch is a separate threading site. Missing either branch of the sign check would leave OAVG (or any user-declared inverse category) vulnerable to false `rotostats_warning_unexpected_slope_sign` fires after the rank-flip correctly produces a negative slope — this was called out as the highest-risk integration point in both spec.md and impact.md.

5. **`INVERSE_CATEGORIES` constant deleted (not kept as a fallback)**: Keeping a dead constant with the same value as the new argument default would invite drift: a future developer might update one without updating the other, creating a hidden discrepancy. Deleting the constant is the only way to make the codebase unambiguous. Builder confirmed via post-edit grep that zero `INVERSE_CATEGORIES` occurrences remain in R source. Architecture documentation (ARCHITECTURE.md) was updated in this scriber dispatch to remove all references to the old constant.

## Handoff Notes

**Key patterns introduced in this run:**

1. **`missing()` detection at function entry**: `inverse_categories_is_default <- missing(inverse_categories)` is the first line of `sgp_denominators()`. This line must remain first if the function is ever refactored — moving it after any code that references `inverse_categories` would break the flag (R evaluates `missing()` based on whether the promise has been forced, not based on position). If a future developer adds a new argument that also uses a default-vs-explicit distinction, this same `missing()` pattern should be applied at the top of the function for each such argument.

2. **`.frequency_id` scheme for `cli_inform()`**: The id `"rotostats_sgp_denom_inverse_" + sorted comma-joined categories` is content-addressed and session-scoped. If `inverse_categories` is later stored in `$meta` on the return object (currently intentionally deferred per spec §7), the `.frequency_id` scheme should be kept in sync so that re-calling with the same configuration stays quiet.

3. **Three substitution sites in `R/sgp-denominators.R` (now `inverse_categories`)**:
   - Line ~676 post-edit: main year-loop `standings_pos` assignment.
   - Line ~783 post-edit: slope-sign check, inverse branch (`cat %in% inverse_categories && beta_c > 0`).
   - Line ~792 post-edit: slope-sign check, normal branch (`!(cat %in% inverse_categories) && beta_c < 0`).
   - Line ~939 post-edit: bootstrap resampling `sp` assignment.
   
   If the bootstrap resampling path is ever refactored (e.g., extracted into a helper), the `inverse_categories` argument must be threaded through or closure-captured.

4. **`INVERSE_CATEGORIES` constant is GONE**: The string `INVERSE_CATEGORIES` no longer exists anywhere in the R source. Architecture docs, planning docs, and specs were updated. Any future reference to this constant in code review comments or documentation should be treated as referring to the deleted object, not a live symbol.

5. **Test section 13 at T-58 through T-70**: The new tests live in `tests/testthat/test-sgp-denominators.R` under the `# === 13. inverse_categories Scenarios ===` header. The `make_oavg_monotone_history()` helper is defined locally in that section and produces deterministic monotone data (OAVG 0.220–0.320, HR 150–230, years 2021–2023). T-25 (ERA regression anchor) lives in section 3 of the same file at T-25 and was re-verified in T-67 (T-NEW-10 guard in section 13).

6. **`$meta` does NOT contain `inverse_categories`**: Per spec §7, the return object's `$meta` slot was deliberately not extended. This avoids breaking callers who enumerate `meta` fields. If a future spec adds `inverse_categories` to `$meta`, it must be done as a backward-compatible addition (existing callers must not break).

**Known limitations:**

- The `cli_inform()` message fires once per unique `inverse_categories` configuration per session. In long-running analyses with many distinct configurations, the rlang frequency cache accumulates entries. This is not a bug but is worth noting if memory becomes a concern in extreme scenarios.
- `inverse_categories = character(0)` is legal (no rank flip applied to any category). This is the correct behavior for all-counting-stat leagues. The inform message for this case says "No categories will be direction-flipped."
