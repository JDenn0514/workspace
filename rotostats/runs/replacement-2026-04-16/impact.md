## Risk Areas

1. **Iteration loop correctness.** The spec states `replacement_level()`
   manages its own convergence loop — caller never orchestrates. When
   `sort_by = "sgp"`, the loop calls `sgp()` internally each pass. Builder
   must handle: stripping `attr(result, "projections")` before the loop and
   re-attaching after convergence to avoid per-iteration copy; respecting
   `sgp_denominators` as fixed (not recomputed per pass); dual convergence
   criteria (zero assignment changes AND `max(|Δrepl_stats|) < tol`).

2. **Zero-sum assertion invariants.** Violation = internal bug, not user
   error — aborts with `rotostats_error_zero_sum_violation`. The
   `zero_sum_positions` set depends on `catcher_adjustment_method`. Tester
   must assert the invariant holds across all four values
   (`split_pool`, `positional_default`, `partial_offset`, `none`).

3. **SP/RP sort-circularity.** Inferred role via `sp_ip_threshold` is a
   documented circular dependency (Q1 #3 in spec). Swingmen (80 ≤ IP ≤ 120)
   are flagged in `cliff_metric$swingman`; builder must compute swingman
   set BEFORE role classification so a pitcher later reclassified by pool
   membership still carries the flag.

4. **Rate-method × denominator-type incompatibility.**
   `boundary_rate_method = "sgp_pool"` + `attr(sgp_denominators,
   "rate_conversion") == "fixed_baseline"` must abort with
   `rotostats_error_rate_method_mismatch` naming BOTH methods. Builder must
   check the attribute, not re-derive.

5. **Thin-pool edge cases.** AL-only 12-team SS gives `n_rostered_pos = 12`,
   dynamic cap `K_eff = min(3, floor(12/4)) = 3` — band of 7 out of 12 (58%).
   The dynamic cap alone does NOT fix this — cliff detection + the `<
   cliff_min_n` skip is the second guard. Simulator must cover this case.

6. **`replacement_from_prices()` independence.** Shares the output contract
   but NOT the algorithm. No `projections`, no `sort_by`, no iteration, no
   cliff detection, no SP/RP inference. Builder must keep the two
   functions' interfaces strictly separate — shared code lives only in
   `R/replacement_internal.R` helpers.

7. **`stat_units` attribute propagation.** Downstream `sgp()` will abort
   with `rotostats_error_stat_units_mismatch` if passed a
   `normalize_to_season = TRUE` result. That downstream guard is OUT OF
   SCOPE for this run — it belongs to a future `sgp()` update. Builder
   should set the attribute correctly but need not modify `sgp()`.

8. **Output attributes carry `projections` (potentially large).** Spec
   notes this explicitly — the convergence loop must strip and re-attach.
   Reviewer should verify no per-iteration copy is introduced.

## Acceptance Criteria

See `request.md` § Acceptance Criteria (three specific criteria) and §
Hard Constraints. The three load-bearing invariants:

1. Boundary band with dynamic K cap + AB-weighted AVG + IP-weighted ERA/WHIP
   + cliff truncation in lower half only.
2. Zero-sum positional-adjustment invariant holds to 1e-6 across all
   `catcher_adjustment_method` values; hitter/pitcher pools separated.
3. SP/RP always separated with role inference + swingman flagging; pitcher
   replacement always includes IP.
