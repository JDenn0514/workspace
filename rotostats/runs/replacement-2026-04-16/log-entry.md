### Proposal Summary

**From `spec.md`:** `replacement_level()` is a per-position estimator using a
symmetric boundary band of `2K+1` players around the `n_teams × roster_slots[pos]`
roster boundary. Key design choices: dynamic K cap (`K_eff = min(K, floor(b/4))`),
cliff detection in lower half only, IP-weighted ERA/WHIP, AB-weighted AVG,
zero-sum enforced as `abs(sum(slots × premium)) < 1e-6` (internal assertion,
never user error), swingman flag computed before role classification,
`projections` attribute stripped before the convergence loop. Four
`catcher_adjustment_method` values; "C" excluded from zero-sum when
`"split_pool"` (default). Multi-position loop terminates on both assignment
stability and `max(|Δstats|) < tol`.

**From `test-spec.md`:** 58 test scenarios (TS-01 through TS-58) covering output
contract, band arithmetic (counting/IP-weighted/AB-weighted), dynamic K cap,
cliff detection (lower-half-only guard), zero-sum across all 4 catcher methods,
SP/RP separation and swingman flagging, all 14 error/warning classes, convergence
behavior, `replacement_from_prices()` including name normalization. Plus 5
simulation validation assertions (SV-01 through SV-05).

**From `sim-spec.md`:** 5 studies (R=500 each), master seed 20260416. Study A:
boundary-band variance stability (K=3 vs K=1). Study B: zero-sum invariant across
all 4 catcher methods. Study C: multi-position convergence under 60%-multi-eligible
pool. Study D: dynamic K cap correctness (12-team AL SS/C and 5-team SS). Study E:
rank invariance of fixed-quality pitcher across 10-team vs 15-team leagues.

### Implementation Notes

**BUG-1 (builder commit 493c4c2 — cherry-pick of b02809b):** All four premium
methods in `compute_positional_adjustments()` iterated over every scored category
regardless of whether the position group had data. Hitter positions have `NA` for
ERA/WHIP; `colSums(na.rm=TRUE)` on an all-NA column returns `0.0`; `NA - 0 = NA`;
`mean(c(NA,NA), na.rm=TRUE) = NaN` (R behavior: mean of empty-after-removal vector).
NaN propagated to all hitter positional adjustments, silently bypassed the zero-sum
assertion (`abs(NaN) > 1e-6 == FALSE`), and caused convergence to never succeed.
Fix: `valid_cats <- scored_cats[!is.na(pos_stats[scored_cats])]` before any premium
arithmetic.

**Secondary fix (same commit):** The `"none"` catcher method recentered using all
of `primary_hitter_slots` (including "C"), then zeroed `scarcity_premium["C"]`.
This broke zero-sum because the recentering was computed over a set that included C
before C was zeroed. Fix: exclude "C" from recentering for `"none"` (same as
`"split_pool"`).

**BUG-2 (same commit):** Step-32 rate-stat guard used a hardcoded category list
that omitted BABIP. Fix: dynamic lookup against
`toupper(names(RATE_STAT_DENOMINATORS))`; BABIP added to `RATE_STAT_DENOMINATORS`
with denominator `"AB"`.

**Simulator DGP fix (commit f5bc595 — cherry-pick of 61a439a):** DGP-A/C/D/E
generated `name` instead of `player_name`, omitted `league` column, used `/`
instead of `|` as `pos_eligibility` delimiter. All four DGPs corrected; pool
sizes increased to satisfy `rotostats_error_pool_too_small` thresholds.

**2-cycle detection (builder commit 21270fb):** Greedy simultaneous reassignment
creates 2-cycles in pools with many multi-eligible players at the same positional
boundary. Added `old_old_assignments` (2-lag state) to the convergence loop;
when `new_assignments == old_old_assignments`, accept as converged. Raised Study C
convergence_rate from 0.002 to 0.966.

**DGP-E focal pitcher fix (same commit, pipeline-isolation crossover):** Focal
pitcher was set above pool mean (ERA=3.50 vs pool 3.80), causing ~75% of the extra
30 SP to rank below the focal pitcher — opposite of the rank-invariance intent.
Changed to boundary quality (ERA=4.70, WHIP=1.40, IP=165). The algorithm was
already correct; root cause was DGP design, not implementation. Builder touched
simulator-owned code (`dgp_e.R`) because the fix was straightforward and a separate
simulator respawn was not warranted. Flagged for reviewer acknowledgment.

**Deferred features (validation guards in place):**
- `seed_method = "historical_priors"` — falls through to primary-position seed
- `boundary_rate_method = "sgp_pool"` — passes through to z-score ranking; `fixed_baseline` incompatibility guard active
- `multi_pos = "all"` — falls back to primary-position assignment

## Design Decisions

1. **`K_eff = min(K, floor(b/4))`** prevents the band from covering more than
   ~50% of any pool regardless of depth, while still providing smoothing in
   typical 12-team pools (K_eff = 3, band = 7).

2. **IP-weighted ERA/WHIP, AB-weighted AVG** produce correct contribution-weighted
   means rather than misleadingly flat means of individual player rate-stat values.
   The spec was explicit; the implementation matches.

3. **Cliff detection in lower half only** preserves the upper half of the band
   (players above the boundary) which is known to be rostered and thus reliable.
   Only the sub-boundary fringe is at risk of a talent cliff.

4. **`catcher_adjustment_method`** controls whether catchers are subject to the
   same zero-sum constraint as other hitters. Default `"split_pool"` gives catchers
   their own sub-pool premium and excludes them from the positional zero-sum
   assertion, matching the standard NFBC/LABR practice of treating catching scarcity
   as distinct from infield/outfield scarcity.

5. **`"none"` catcher treatment requires same recentering exclusion as `"split_pool"`**
   (discovered as a latent bug when BUG-1 was fixed). The pattern: apply catcher
   override AFTER recentering, not BEFORE — and always exclude "C" from the
   recentering set when the override sets C's premium to something other than
   the globally-derived value.

6. **`stored_projections` pattern** for the iteration loop: strip attribute before
   loop, reattach after convergence. Prevents large projections data frame from being
   copied on each of `max_iter = 25` passes.

7. **BABIP = "AB" added to `RATE_STAT_DENOMINATORS`** because BABIP is a valid
   rotisserie category and its denominator (AB) is unambiguous. The step-32 guard
   now uses the live lookup table rather than a hardcoded list, ensuring any future
   additions to the lookup are automatically covered.

## Handoff Notes

1. **Study C follow-up ticket (recommended):** Higher-order cycle detection in the
   `highest_par` loop. Current 2-lag (`old_old_assignments`) handles the common case
   but misses 3-cycles+ in stress pools. Approach: hash the position assignment state
   (e.g., `digest::digest(new_assignments)`) and check against a small history window.
   Expected to push convergence_rate from 96.6% → 99%+.

2. **Study E follow-up ticket (recommended):** Either tighten DGP-E further (focal
   pitcher further below pool mean to minimize pool composition variance effects) or
   review the acceptance thresholds (median ≤ 5 and pct_within_2 ≥ 0.80 may be more
   appropriate for a static focal pitcher). The algorithm itself is correct (Study D
   confirms boundary indexing).

3. **Deferred features:** Three features have validation guards but incomplete
   implementations: `seed_method = "historical_priors"`, `boundary_rate_method = "sgp_pool"`,
   `multi_pos = "all"`. Each needs a sub-spec before implementation. Do not silently
   remove the validation guards.

4. **`sort_by = "sgp"` with `blended_pool` denominators requires `league_history`:**
   When `replacement_level()` calls `sgp()` internally on pass 2+, it forwards
   `league_history` (if supplied). If the user passes `sgp_denominators` calibrated
   with `rate_conversion = "blended_pool"` but no `league_history`, `sgp()` will abort.
   This constraint is documented in the roxygen `@param sgp_denominators` description;
   downstream `dollar_values()` should always forward `league_history`.

5. **Pipeline-isolation crossover `dgp_e.R`:** Builder touched simulator-owned code
   in `21270fb`. If the simulator is respawned for a future sim run, ensure DGP-E's
   `.FOCAL_PITCHER` constants (ERA=4.70, WHIP=1.40, IP=165, W=8, K=130) and the
   calibration comment are preserved.

6. **`devtools::document()` required after any roxygen2 change** in
   `R/replacement.R`, `R/replacement_internal.R`, or `R/replacement_params.R`.
   The current run produced clean doc generation.
